## Part 1 — 第一性:从欲望与约束推导内核

### 1.1 真正的欲望

把品类标签(「AI 角色扮演」「AI 文字冒险」)全部拿掉,剩下的人类欲望只有
一句:

> **走进另一个世界,并且那个世界是「真的」——它因我而变,记得我,会让我意外。**

这是最古老的欲望,小说、戏剧、电子游戏都在满足它的一部分。关键在「真」这个字
拆成三个可操作的子诉求:

1. **因果**:我做的事有后果,后果可预期地累积。
2. **持久**:我离开再回来,世界没有重置。
3. **意外**:世界有我不知道的部分,会反过来给我惊喜(而不是我编、它附和)。

### 1.2 三种已有形态各缺一角

| 形态 | 因果 | 持久 | 意外 | 反应深度 | 缺的那一角 |
|---|---|---|---|---|---|
| 书 / 线性小说 | ✓(作者写死) | ✓ | ✓ | **零**(不能交互) | 没有「我」 |
| 电子游戏 | ✓ | ✓ | 部分 | **有限**(反应要人手预制,贵→浅) | 反应性被作者成本封顶 |
| AI 聊天 / 角色扮演 | **✗** | **✗** | ✓ | 无限 | 世界在两条消息之间**不存在** |

AI 聊天第一次给了**无限反应性**,但代价是**结构性真实的彻底缺失**:它没有状态,
所以打翻的杯子下一句就忘、拿走的东西没人在意、锁着的门可以靠嘴遁穿过去。它把
「世界」等同于「模型这一句的输出」。

### 1.3 载荷性洞察:LLM 是绝佳的渲染层,糟糕的数据库

这是整个产品唯一真正不可替换的判断:

> **LLM 擅长把状态渲染成有质感的散文、把自然语言解析成意图;它不擅长「是」
> 一个一致的世界。** 让它**当**世界(纯聊天)→ 无限反应性、零真实。让它**渲染**
> 世界(状态机在下)→ 反应性几乎不损失,而真实(因果/持久/一致)被状态机
> 兜住。

由此得到内核第一条,其余一切都是它的推论:

> **世界 = 一层结构化、可校验的状态机。LLM 只是它的渲染层(输出)和玩家的
> 输入层。模型永不直接改世界——它只「提议」经校验的结构化变化,引擎按不可变
> 规则核准后才落库。**

### 1.4 为什么押注「文字」是被逼出来的,不是情怀

承上:产品的核心价值是**反应深度**(任意角色、任意分支、任意后果都成立)。在
所有交互媒介里,反应深度的瓶颈是**「每多一种反应」的边际成本**。

- 配音 / 3A 渲染的 NPC:每多一句反应 = 一次录音 / 一段动画 → 边际成本高 →
  反应被迫封顶在浅层(两三行对白、几个预设分支)。
- 文字:每多一种反应 = 更多 token → 边际成本趋近于零。

所以**文字不是审美偏好,是「无限反应深度」这个核心价值在当前技术下唯一付得起
的载体。** 「文字只是交互形式,不是世界本质」这句 slogan,本质是在说:别把成本
花在像素上,花在反应性上。

### 1.5 三个工程约束,各逼出一个产品特征 —— 而且都与「真」同向

这是本次推导最重要的发现,也是这个产品能成立的根本理由:

| 硬约束(物理) | 逼出的产品特征 | 它**同时**也让世界更「真」 |
|---|---|---|
| **每回合都要花真金白银调 LLM**(推理成本) | **BYO-key + local-first** & **Consequence Mode**(离开即冻结,回来懒补) | 平台零空转 → 隐私归用户、可不设限;「不实时模拟、回来才补」恰好就是「世界没在背后空转却仍合理演化」的体感 |
| **context window 有限**,装不下 GTA 级全量世界 | **ambient-by-default / earned persistence**(只结晶被注意到的部分) | 「新细节像本来就在,你只是刚走近」——这正是渐进展开的美学 |
| **LLM 会幻觉、会自相矛盾、会越权** | **propose → validate → apply 闸门** & **角色只收主观投影** | 引用不存在之物=非法 delta 被丢 → 永不自相矛盾;角色只知道自己见证的 → 信息差与戏剧反讽 |

> **关键结论:工程约束与产品美学在这个设计里是同向的。** 省钱的做法恰好最有
> 沉浸感,省 context 的做法恰好最像真实世界的渐进展开,防幻觉的做法恰好制造
> 戏剧。一个设计若约束与美学同向,通常说明它抓对了第一性。**反过来,凡是二者
> 不同向的地方,就是真正的风险点(见 Part 4)。**

### 1.6 内核(不可动摇,均为上面推论)

1. **世界是状态机,文字是其交互层。**(1.3)
2. **模型提议,引擎校验,校验后才落库。**(1.3)
3. **门只属于玩家。** 玩家是唯一从世界外推门进来的实体;其余一切都是世界
   自身展开、细化出来的。(这是「世界是源头」的人格化表达,保证导入的角色/
   新细节都不破坏「世界自洽」。)
4. **默认 ambient,挣得持久才结晶。**(1.5 context 约束)
5. **角色只收主观投影,永不读裸状态;Director/Reactor 全知但不是角色。**
   (1.5 防幻觉约束 + 戏剧需要信息差)
6. **交互驱动,离开即冻结,回来懒补(Consequence Mode 默认)。**
   (1.5 成本约束)
7. **本地优先、自带 key、可不设限。**(1.5 成本约束 + 6 的隐私结果)

这 7 条与 `AGENTS.md` §2/§13 完全一致——区别只在:本文是把它们**推**出来,
而非声明出来。

## Part 2 — 产品设计:作为内核的推论

### 2.1 北极星与最难的承诺

> **私人活体世界浏览器**:竖滑无数门,推门即入,用文字活在里面;开过的门
> 不是一次性聊天,而是会越来越真、越来越懂你的私人世界实例。

最难承诺(产品的全部赌注):

> **每一扇门都一眼就想推开;每一个你愿意留下的世界,都越待越真。**

注意这是**两个**承诺,服务**两端漏斗**:前半句是**获取**(秒级吸引),后半句
是**留存**(长期变真)。产品的所有动作都应能归到这两端之一。

### 2.2 漏斗 = 一串「证明义务」

把产品拆成阶段,每个阶段有一个**必须当场证明给用户看**的命题。设计的纪律就是
「这一段到底证明了什么」。

| 阶段 | 用户在干嘛 | 必须证明的命题 | 由谁兑现 |
|---|---|---|---|
| **发现 Discover** | 竖滑 feed | 「这扇门值得推」——秒级可判、勾人、不重复 | 口味引擎 + 冷开场卡 + 世界生成器 |
| **跨阈 Threshold** | 推门转场 | 「我真的从外面走进了一个里面」——仪式感 | 开门转场 + 第二人称冷开场 |
| **头十分钟 Proof** | 头几个回合 | 「这是个真世界」:① WorldState 有东西因我而变;② 有人记得 / 误记我;③ 一条本地后果出现;④ 至少一条 canon 被挣得 | 回合循环(Reactor/记忆/flesh) |
| **回访 Return** | 隔天再开 | 「世界在我不在时合理地走了一点,且没重置」 | Consequence Mode 懒补 + Doorway Library echo |
| **加深 Deepen** | 长玩同一世界 | 「它越来越懂这个世界里的我」:关系账本、秘密、悬念、anchor 累积 | 关系账本 + 主观记忆 + pressure lines + Home/Anchor |
| **掌控 Power** | 高阶用户 | 「我能导演 / 修正 / 分叉,而不戳破真实」 | Director Notes / Scene Contract / God Mode / Atlas / Forks |

设计评审就一句话:**任何新功能,先说它兑现的是哪一行的命题。** 兑现不了某一行
的功能,默认不做(对齐 ROADMAP 原则一「先让世界更真,再让功能更多」)。

### 2.3 各产品面如何映射到漏斗

- **Feed(发现)**:门缝而非百科。门名 + 一句冷开场 + 调性/烈度 + 一条未决张力
  + 一个显然的开门动作。**不把 tag 当主推荐面**——tag 是内部信号,用户感知到的
  是门、钩子、活的可能性。
- **Play(头十分钟 + 加深)**:lean-in 在场。本地场景 + 可交互角色/物/痕迹 +
  一条即时张力 + 玩家输入(说/做/观察/导演)+ **针对你的具体动作的可见后果**。
- **Doorway Library(回访)**:开过的门是持久私有实例,不是废弃聊天。带上次
  位置、未决张力、最新后果、关系状态;轻 echo,不做激进推送。
- **控制层(掌控)**:默认沉浸 Player Mode;Director Notes / Scene Contract /
  God Mode 按需浮现。**频道隔离**是铁律(见 2.4)。
- **口味编年史(发现的引擎)**:学**行为序列**而非 tag(停留/扎根/快划/首动作/
  分叉)。生成三档:利用 50% / 桥接 35% / 探索 15%。**桥接是招牌**:保留深层
  吸引结构、只换表层题材/角色/权力关系,避免信息茧房。

### 2.4 两类用户如何不撕裂成两个产品

入门沉浸玩家(零概念)与深度 RP/NSFW/创作 power user(要控制/分叉/边界/慢热),
靠**单向可发现性 + 频道隔离**共存:

- **默认面永远是沉浸**;所有高级面是**藏起来、可被发现**,绝不默认占据 play 界面。
- **频道隔离是不可妥协的纪律**:Player Mode / Director Notes / Scene Contract /
  God Mode 互不串味。角色**不会**自动知道导演笔记、场景契约、跨世界口味历史,
  除非世界显式把它 canon 化。
- 这条纪律是一种**持续的税**(每加一个 power 面都要重新确认它不漏进角色知识),
  Part 4 会指出它是长期张力之一。

## Part 3 — 技术方案:作为产品的推论

### 3.1 世界的工程定义

```text
世界 = WorldRules(不可变) + WorldState(可变快照) + 校验过的 Delta 日志(追加式) + 各 agent 主观记录
```

- **WorldRules** 创建后只读:physics / setting / redLines。
- **WorldState** 可变且按需生长:地点/物体/roster/角色/关系/lore/flags/tension/time。
- **Delta 日志**是历史真相:每条经校验落库的 delta 记 turn / 世界时间 / 来源 /
  触发它的输入。延时回调、世界声誉、离场演化都从它长出来。
- **主观记录**:每个角色自己的观察/记忆/信念/秘密/反思,带 provenance。

> 这正是内核第 1、2 条的数据化:LLM 渲染上面这坨,但只能通过 delta 闸门改它。

### 3.2 选定架构:turn-scoped layered runtime

**不建常驻服务端模拟。** 世界在用户交互时推进,回来时懒补离场后果。理由全部
来自 1.5 的成本约束 + 6/7 的本地优先/BYO-key。未来 Living World Mode 可复用同
一批模块换一个更主动的调度器,但**不是默认架构**。

模块边界(职责以 world-runtime spec 为准,此处给推导关系):

| 模块 | 职责 | 它在兑现内核哪条 |
|---|---|---|
| **TurnOrchestrator** | 单回合生命周期:锁/快照/路由/编排/落库/回滚 | 闸门的事务性 |
| **InputRouter** | 把输入分流为 speech/action/observe/storyIntent/directorNote/godEdit | 频道隔离(2.4) |
| **WorldKernel** | **唯一**durable-state 写入者:validate→apply→log→记录 rejected | 内核 2(提议/校验/落库) |
| **ContextAssembler** | 给每个角色产出**有预算的 typed context pack**(借 SillyTavern world-info 机制:预算/递归/sticky/cooldown) | context 约束(1.5) |
| **PerceptionResolver** | 构造每个角色的 `SubjectiveContext`,带 provenance | 内核 5(主观投影) |
| **Director** | 全知编排:选 pressure、选本回合 active agents、决定是否/如何浮现潜伏实体、把新细节框成世界原生 | 内核 3、4;ambient 上限 |
| **AgentRuntime** | 跑被选中的(≤ ~4)角色:perceive→retrieve→intent→speak/act→observe | 内核 5;成本(capped) |
| **Reactor** | 客观后果翻译器:读回合 trace → 提议 `Delta[]`,**证据优先** | 内核 2 的提议端 |
| **Materializer** | 把挣得持久的 ambient 细节结晶成结构实体(stub→fleshed + establish*) | 内核 4 |
| **MemoryBeliefSystem** | 观察/二手/信念/秘密/反思/关系证据,各自可分辨 | 内核 5;社会因果 |
| **OffstageReconciler** | Consequence Mode:回来时按离开时长提议有界的小 delta,只给迹象不搞重大不可逆 | 内核 6 |
| **TasteSeedRuntime** | 世界生成在世界实例之外;口味只影响新门,不漏进角色知识 | 2.3 口味;频道隔离 |
| **StudioRuntime** | Director Notes / Scene Contract / God Mode / Atlas / Inspector / Forks,频道隔离 | 2.4;掌控段 |

### 3.3 全回合流(目标)

```text
取实例锁
→ 载入实例/seed/近期消息/记忆游标
→ 拍回滚快照
→ 分类输入频道(InputRouter)
→ 经 WorldKernel 应用显式 user/god delta
→ 若回访:OffstageReconciler 懒补
→ Materializer 做感知前必需的 flesh
→ Director 选 pressure / active agents / ambient cast / 是否浮现
→ PerceptionResolver 为每个 active agent 建 SubjectiveContext
→ AgentRuntime 跑被选角色(流式发言)
→ Director 渲染 ambient cast / 旁白 beat
→ Reactor 提议 Delta[]
→ WorldKernel 校验/应用/落库
→ MemoryBeliefSystem 写观察/信念/二手/反思
→ Materializer 后置 flesh 新挣得的实体
→ 持久化 实例/消息/记忆/deltaLog
→ 释放锁(失败则恢复快照、清理本回合产物)
```

并发纪律(AI Town 式):一实例同时只允许一个回合落库;每个长 LLM 操作有
operation id;过期结果丢弃;超时安全释放锁;重生成/分叉/god edit 作废在途操作。
**加更多 agent 调用之前必须先有这套锁**,否则并行的 Director/Reactor/角色结果会
污染同一私有分支。

### 3.4 现状 vs 目标(诚实对账)

`docs/DESIGN.md` 显示**地基已相当完整**,但多数职责仍集中在单体 `runTurn` 里:

- **已实现**:vertical feed/冷开场/转场/play;BYO-key + IndexedDB;
  `WorldRules+WorldState+Delta` 模型;`reactor→validate→apply` 闸门;
  追加式 deltaLog;地点连通 + 锁/gate/portable 物理约束;lore 注入 +
  递归级联 + establishLore;主观记忆 + 反思 + 关系账本 + 二手传话;
  地点 stub→fleshed 首访充实;establishCharacter + 实例私有角色;
  evolveWhileAway 离场懒补;重生成上一条。
- **还没拆出来**:WorldKernel / ContextAssembler / PerceptionResolver /
  Director-as-caster(active-agent cap + 主动浮现)/ 显式 AgentRuntime 认知步 /
  分类化的 MemoryBelief / pressure lines 的结构化 / Studio instrumentation。

> **结论:这不是从零造,而是「从单体 `runTurn` 向外拆模块」。** 不要另起一套
> 竞争性的循环;按 spec 的 slice 顺序把边界一段段抽出来,语义保持不变。

### 3.5 实施顺序(对齐 runtime spec 的 slices)

1. **抽 WorldKernel**:把 validate/apply/log 收到 `commitDeltas` 后面,加
   rejected 记录(给 Inspector 用),行为不变。
2. **ContextAssembler + PerceptionResolver**:用 typed pack 取代 prompt 拼接,
   加测试**证明** POV 边界(角色看不到私有/远程/全知事实)。
3. **Director Casting**:Director 选 active/ambient,落 `maxActiveAgents`,
   删掉「tension≥6 抓第一个幕后角色」的硬编码。
4. **AgentRuntime 显式认知步**:perceive→retrieve→intent→speak/act→observe,
   回合末仍由 Reactor 做结构写。
5. **Memory/Belief 升级**:观察/二手/信念/秘密/反思分型 + provenance + confidence,
   关系证据连到记忆记录。
6. **Offstage + Pressure Lines**:pressure line 进 seed/state;离场懒补读 pressure
   与证据;后果只走 diegetic 迹象。
7. **Studio instrumentation**:为 Context Inspector 暴露 trace;Director Notes /
   Scene Contract / God Edit 走频道隔离路径。

每一 slice 都**先写证明边界的测试再动实现**(对齐现有 TDD 习惯与历史 plan 风格)。

## Part 4 — 被低估的硬问题(约束与美学**不**同向之处)

1.5 说约束与美学多数同向。下面这些是**不同向**的地方——locked specs 提了一两句
或没提,但它们决定产品成败,应被当作一等公民排期。

### 4.1 渲染 / 状态 一致性裂缝(最致命)

**问题**:闸门只保护**durable state**。但用户**读到的散文**不经校验。角色完全
可以**嘴上**说「门没锁」,而 state 里门是锁的;Reactor 可能没把这句翻译成 delta,
于是用户读到的世界与真实 state 悄悄分叉。整个「真」的承诺,在「散文层」这里有
一条结构性漏洞:**state 是真相,但用户体验的是 prose,而 prose 可以撒谎。**

**方向**:
- 把散文**接地(grounding)**到主观投影:发言/旁白的 prompt 不只给「能说什么」,
  也给「当前事实快照」,并明确指示「不得断言与【此刻所见】矛盾的物理事实」。
- 引入**轻量一致性后检**(可选、可降级):回合末用一次便宜调用或规则检查,
  扫散文里对**已结构化物体/门/位置**的硬断言,与 state 对不上就标记/软修。
- 接受**分层真相**:把 prose 显式定义为「角色的**主张**」,只有 state 是「世界的
  **事实**」。主张可以错(这恰恰也是戏剧),但**物理硬事实**(锁/在不在场/位置)
  不允许 prose 单方面翻案。这条边界要写进 prompt 契约与测试。

### 4.2 延迟预算(直接对撞「沉浸」)

**问题**:一个回合 = 离场懒补 + N 路并行 intent + N 个流式发言 + Director +
Reactor + 记忆 + flesh,**多次串行 LLM 调用,跑在用户自己(可能很慢/很便宜)的
key 上**。沉浸要求即时,几十秒级回合直接杀死沉浸。spec 几乎没给延迟预算。

**方向**:
- 设一个**回合延迟预算**(如:首 token < 1.5s,完整结算 < 回合可接受上限),
  把模块按「是否阻塞用户读到第一段散文」分成**快路径 / 慢路径**。
- 快路径:Director 选角 + 最相关角色的流式发言**先出**,让用户立刻有东西读。
- 慢路径(可在散文流式时并行 / 之后结算):Reactor 落 delta、记忆写入、二手传话、
  后置 flesh。这些不阻塞「读到回应」。
- 给 Reactor / 记忆这类**结构写**留一个「回合末批处理」窗口,容忍它比散文晚到。

### 4.3 每回合成本(跑在用户钱包上)

**问题**:全管线每回合多次调用,用户自带 key 即用户自掏腰包。`maxActiveAgents≈4`
有帮助,但 Director+Reactor+记忆+flesh 仍是固定开销。深度与花费正相关。

**方向**:
- 给世界/全局一个**深度档**(快而省 / 标准 / 深):档位调的是 active agent 上限、
  是否每回合跑 Reactor、记忆/反思频率、flesh 触发阈值。
- **合并调用**:intent 这类轻判断可批量进一次调用;低风险回合可跳过 Reactor
  (无结构变化的纯对话回合不必每次都翻译 delta)。
- 把「这回合花了几次调用」做成 Inspector 可见,让 power user 自己权衡。

### 4.4 Reactor 的精度 / 召回(承载「真」的那一次调用)

**问题**:整个「世界会变」全压在 Reactor 把散文→delta 翻译对。**欠提议**→世界
显得静止(后果没落库);**过提议**→幻觉出没真发生的状态变化。这是个没有完美解
的 precision/recall 问题,却是最 load-bearing 的一次 LLM 调用。

**方向**:
- 维持**证据优先**(只为近期散文里确已发生的事提议,不为被提及/打算/假设的事写)
  并把它做成可测断言。
- 借 Voyager 式 **critic 回路**:Reactor 不只生成 delta,还要让每条 delta
  **可检验**(附它依据的那句散文证据);WorldKernel 拒绝时记下理由供 Inspector 看。
- 对**高后果 delta**(关系大幅变动、上锁、角色生灭)可设更严的二次确认;
  对低后果 delta 放行从宽。

### 4.5 Feed 冷启动成本与质量(漏斗最上游)

**问题**:整个漏斗从 feed 开始,而**生成好门本身就是一串 LLM 调用**。BYO-key
意味着「光是浏览也要用户付费 / 配 key」,这是获取期的摩擦。locked default 已给
解法(无 key 可浏览静态示例门,实时生成/游玩才要 key),但 feed 生成的成本与
质量不归 runtime spec 管,需要单独的拥有者。

**方向**:
- **静态示例门池**做足做精,保证无 key 首屏就惊艳(获取段的「这扇门值得推」由
  它兜底)。
- 实时生成走**后台预生成池 + 多样性约束**,均摊延迟;首跑跨题材铺开解决冷启动。
- 明确把「feed 生成质量」当成与 runtime 并列的一等子系统,有自己的评测信号
  (秒级可判率、推门转化率、扎根率)。

### 4.6 两类用户的持续张力(频道隔离的长期税)

**问题**:2.4 的频道隔离是对的,但它是**每加一个 power 面都要重交的税**——任一
处疏忽(导演笔记漏进角色 prompt、跨世界口味漏进角色知识)都直接戳破沉浸,而且
这类 bug **静默**(不报错,只是世界「不该知道却知道」)。

**方向**:
- 把「角色 prompt 里**绝不**出现 directorNote / sceneContract / 跨世界 taste /
  god-edit 未 canon 化内容」做成**常驻测试断言**,而非靠人记。
- PerceptionResolver 作为**唯一**喂给角色的入口,所有角色上下文必须穿过它——
  这样隔离只需在一处守住,而不是 N 处。

这些是品味/范围选择(推不出唯一答案),建议显式拍板:

2. **延迟优先级**(4.2):是否把「首 token 速度」提为与「世界真实」并列的一等

3. **深度档**(4.3):快/标/深三档是否进 MVP,还是先单档?(建议:MVP 单档跑通,
