## Part 0 — 这份文档想回答的问题

现有文档大多是「结论 + 论证为什么这个结论对」。本文反过来:**只假设最少的前提
(人类欲望 + 感知的本性 + 物理约束),看产品和架构能不能作为推论自然长出来。**
如果能,说明设计是「被逼出来的」而非「拍出来的」。本次推导的最大收获,是发现整套
架构可以从**一条公理**(单一真相中枢)生长出来,控制层、God Mode、个性化、感知、
材化、闸门全是它的枝。

## Part 1 — 第一性:从欲望、感知、约束推导内核

### 1.1 真正的欲望

把品类标签(「AI 角色扮演」「AI 文字冒险」)全部拿掉,剩下的欲望只有一句:

> **走进另一个世界,并且那个世界是「真的」——它因我而变,记得我,会让我意外。**

「真」拆成三个可操作子诉求:**因果**(做的事有后果且累积)、**持久**(离开再回来
不重置)、**意外**(世界有我不知道的部分、会反过来给我惊喜,而非我编它附和)。

### 1.2 三种已有形态各缺一角

| 形态 | 因果 | 持久 | 意外 | 反应深度 | 缺的那一角 |
|---|---|---|---|---|---|
| 书 / 线性小说 | ✓(写死) | ✓ | ✓(精校) | **零** | 没有「我」 |
| 电子游戏 | ✓ | ✓ | 部分 | **有限**(预制成本封顶) | 反应性被作者成本封顶 |
| AI 聊天 / 角色扮演 | **✗** | **✗** | ✓(高方差) | 无限 | 世界在两条消息间**不存在** |

AI 聊天第一次给了**无限反应性**,代价是**结构性真实的彻底缺失**。但要看清它到底
缺在哪、该怎么补,得先看「真世界」在感知层面的本性——这就引出最底层那条公理。

### 1.3 最底层公理:单一真相中枢(星形拓扑)

现实里,我能接触到的永远只是**世界的一部分**——我直接感知的那点,加上其他角色;
而每个角色同样只持有自己**部分的**世界认识。这带来一个致命问题:

> **若每个部分感知者都用一个独立的模型、各自去生成/细化现实,他们会发展出彼此
> 矛盾的世界。「多性(multiplicity)」本身就会发散——哪怕每个模型都完美。**

两个角色背对背描述同一个房间,会编出两个房间;一个角色记得的「昨天」和另一个角色
记得的「昨天」会对不上。要让 N 个部分感知者**共处同一个世界而不分叉**,唯一的解是:

> **必须有一个看到完整世界的全知者(god / 导演)持有唯一真相,再把「你感知到了
> 什么」分发给每一个人。** 这是**星形拓扑**:一个真相中枢 + 许多部分辐条;
> **绝不是 N 个独立心智组成的网状。**

这条公理是整套架构的根。它派生出两条箭头,而这两条箭头**就是**整个 runtime:

- **向外(感知 / push)**:中枢 → 每个 agent 的**部分投影**。agent 永不读裸真相。
- **向内(能动 / pull + propose)**:agent 有**主观能动性**——主动**拉信息**
  (观察、探问、与人/物互动)和**提议改变**(行动);二者**全部回到中枢裁决**。
  「凑近看 → 世界当场把那个角落生成出来」(按需材化)也是 pull 触发、**也过中枢**,
  所以两个人看同一个角落,看到的是**同一个**。

**一个推论顺手把后面的控制轴接到了根上**:玩家和角色**在「世内」是对称的**——都是
部分感知,都能 pull/propose,都只能靠**世内行动**过中枢拉信息(角色不能直读全知
真相,否则信息差就破了)。**玩家唯一的不对称,是多了一根「世外权威轴」**(导演
笔记 → God),即 Part 2.4 的控制轴。

### 1.4 推论:LLM 是渲染层与提议者,不是世界本身

中枢持有唯一真相,意味着真相必须是**可被一个权威持有、校验、分发**的东西——也就是
**结构化、可校验的状态机**,而不是「某个模型这一句的输出」。于是:

> **世界 = 一层结构化可校验的状态机(中枢)。LLM 只负责把它渲染成散文、把自然
> 语言解析成提议;模型永不直接改世界,只「提议」,经中枢按不可变规则校验后才落库。**

注意这比「LLM 记不住、所以要外挂数据库」**更耐用**:状态机的必要性来自**多性下的
一致性**,不只来自单模型的记性。即便将来出现一个记忆完美的模型,只要世界由**多个**
部分感知者共享,你仍需要一个单一真相中枢——否则它们仍会发散。而且状态机还顺带是
**控制**的基底(God 改 / Atlas 看 / Fork 分叉 / 红线校验都要求世界可读取、可校验、
可编辑、可分叉)。**所以状态机能挺过模型变强:它服务的是「多性下的一致性 + 用户
掌控」,不是「给笨模型打补丁」。**

### 1.5 为什么押注「文字」是被逼出来的

核心价值是**反应深度**(任意角色、任意分支、任意后果都成立)。它的瓶颈是「每多一种
反应」的边际成本:配音 / 3A 渲染的 NPC 每多一句要录音/做动画 → 成本高 → 反应被迫
封顶在浅层;文字每多一种反应 = 更多 token → 边际成本趋近于零。**文字不是审美偏好,
是「无限反应深度」当前唯一付得起的载体。** 图像/语音作为 render-layer 资产不冲突。

### 1.6 三个工程约束 × 美学:过度决定(over-determined)

| 硬约束(物理) | 逼出的产品特征 | 它**同时**也让世界更「真」 |
|---|---|---|
| 每回合都要花真金白银调 LLM | **BYO-key + local-first** & **Consequence Mode**(离开即冻结,回来懒补) | 平台零空转 → 隐私归用户、可不设限;「不实时模拟、回来才补」恰好是「世界没空转却仍合理演化」的体感 |
| context window 装不下全量世界 | **ambient-by-default / earned persistence** | 「新细节像本来就在,你只是刚走近」的渐进展开美学 |
| LLM 会幻觉/矛盾/越权 | **propose → validate → apply 闸门** & **角色只收主观投影** | 引用不存在之物=非法 delta 被丢 → 永不矛盾;角色只知见证之事 → 信息差与戏剧 |

**关键:不止「同向」,是过度决定。** 比如 ambient-by-default,被三股**独立**的力同时
指向——context 装不下、作者写不起 GTA 级地图、以及「像本来就在」的美学。多股独立的
力收敛到同一批特征,这种收敛才是我们对它有信心的真正原因。

诚实的边界:**同向只在内核层成立;到执行层三力会分叉**(Consequence Mode 是零空转
下的可接受幻觉,和真模拟有残余质量差;延迟与沉浸对撞……)——那正是 Part 4 的战场。

### 1.7 内核(不可动摇,均为上面推论)

1. **单一真相中枢 / 星形拓扑**:一个全知者持有唯一世界,分发部分投影;agent 永不
   读裸真相,只能向中枢 pull/propose。(1.3)
2. **世界是状态机,文字是其渲染/交互层;模型提议,中枢校验,校验后才落库。**(1.4)
3. **门只属于玩家。** 玩家是唯一从世界外推门进来的实体;其余一切都是世界自身展开、
   细化出来的(世界是源头)。
4. **默认 ambient,挣得持久才结晶。**(1.6)
5. **角色只收主观投影(机械事实为底),Director/Reactor 全知但不是角色。**(1.3)
6. **交互驱动,离开即冻结,回来懒补(Consequence Mode 默认)。**(1.6)
7. **本地优先、自带 key、可不设限。**(1.6)

## Part 2 — 产品设计:作为内核的推论

### 2.1 北极星与最难承诺

> **私人活体世界浏览器**:竖滑无数门,推门即入,用文字活在里面;开过的门是会
> 越来越真、越来越懂你的私人世界实例。

最难承诺(两个,服务漏斗两端):

> **每一扇门都一眼就想推开**(获取)**;每一个你愿意留下的世界,都越待越真**(留存)。

### 2.2 漏斗 = 一串「证明义务」

| 阶段 | 用户在干嘛 | 必须当场证明的命题 | 由谁兑现 |
|---|---|---|---|
| **发现** | 竖滑 feed | 「这扇门值得推」——秒级可判、勾人、不重复 | 口味引擎 + 冷开场卡 + 世界生成器 |
| **跨阈** | 推门转场 | 「我真的从外面走进了一个里面」 | 开门转场 + 第二人称冷开场 |
| **头十分钟** | 头几回合 | ① WorldState 有东西因我而变 ② 有人记得/误记我 ③ 一条本地后果 ④ 一条 canon 被挣得 | 回合循环 |
| **回访** | 隔天再开 | 「世界在我不在时合理地走了一点,且没重置」 | Consequence Mode 懒补 + Doorway Library echo |
| **加深** | 长玩同一世界 | 「它越来越懂这个世界里的我」 | 关系账本 + 主观记忆 + pressure lines + anchor |
| **掌控** | 高阶用户 | 「我能导演/修正/分叉,而不戳破真实」 | 控制轴(见 2.4) |

设计纪律一句话:**任何新功能,先说它兑现哪一行命题;兑现不了的默认不做。**

### 2.3 各产品面映射漏斗

- **Feed(发现)**:门缝而非百科;不把 tag 当主推荐面。
- **Play(头十分钟 + 加深)**:lean-in 在场;针对你的具体动作的可见后果。
- **Doorway Library(回访)**:持久私有实例,带上次位置/张力/后果/关系;轻 echo。
- **控制轴(掌控)**:见 2.4。
- **口味编年史(发现的引擎)**:学**行为序列**而非 tag;利用 50% / 桥接 35% /
  探索 15%,桥接为招牌(保留深层吸引结构,只换表层)。

### 2.4 统一控制模型:一根用户权威轴(本设计的枢纽)

控制不是四个并列的「模式」,是**一根连续轴:用户对世界事实的权威有多大(发现 ←→
创作)**。控制层 / God Mode / 个性化全部收编进来。它直接坐在 1.3 的公理上——**玩家
本就是世内对称的 agent,这根轴是他额外的「世外权威旋钮」**。

| 档位 | 能提议什么 delta | 本质 |
|---|---|---|
| **纯玩家 Player** | 散文 → Reactor → delta,且必须 in-world 可信(锁着的门挡你) | **发现**世界 |
| **导演笔记 / 场景契约** | **方向性软 delta**(调 tension / pacing / disposition 朝向 / 世界对你的注意力),频道隔离不进角色知识 | 软操控 |
| **God / Studio** | **任意硬 delta**(改关系数值、retcon、立先前历史),仍走校验 + 落库 + 仅私有分支 | **创作**世界 |

**贯穿全轴的不变量(第一性得以保住的原因):**

> 轴上任何一点,改世界的唯一途径都是「经校验的 delta + 落事件日志」。控制权提升,
> 提升的只是「你能提议哪种 delta」,**绝不是「能否绕过闸门」。闸门永不绕过。**
> 因此**你创作出的世界,和你发现来的世界一样真**(一致 / 持久 / 可审计 / 可分叉)。

这一条解了长期卡住的「真世界 vs 围着我转」之争:**它俩不是对立,是同一根轴的两端**;
realness 不变量(闸门 + 中枢)在两端都成立。想要冷漠的硬核世界,旋钮拧到发现端;
想要角色为我着迷、世界让路,拧到创作端——但他着迷的「凭什么」仍进关系账本、仍受
信息差约束、锁着的门仍挡路。

**白捡的收编:个性化 =「门越来越懂你」= 系统按口味编年史自动预设这根旋钮**(世界
初始顺从度),且**只作用在导演 / God 层,绝不变成角色对你的 in-world 知识**——刚好
守住「口味不漏进角色知识」的老不变量。于是控制层、God Mode、个性化三者收成一个模型。

### 2.5 两类用户如何不撕裂成两个产品

入门沉浸玩家(零概念)与深度 RP/NSFW/创作 power user 靠**单向可发现性 + 频道隔离**
共存:默认面永远沉浸;高级面藏起来、可被发现。**频道隔离是不可妥协的纪律**——角色
不会自动知道导演笔记、场景契约、跨世界口味,除非世界显式 canon 化。这是一种**持续
的税**(见 4.5)。

## Part 3 — 技术方案:作为产品的推论

### 3.1 世界的工程定义

```text
世界 = WorldRules(不可变) + WorldState(可变快照,= 真相中枢) + 校验过的 Delta 日志(追加式) + 各 agent 主观记录(投影)
```

### 3.2 两条箭头 = 整个 runtime

1.3 的两条箭头落到工程上:

- **向外(perception / push)**:`PerceptionResolver` 从中枢算出每个 agent 的
  `SubjectiveContext`(带 provenance)。
- **向内(agency / pull + propose)**:玩家/角色的输入经 `InputRouter` 分流为
  speech / action / **observe(pull)** / storyIntent / directorNote / godEdit;
  pull 触发材化、propose 触发 Reactor,**全部经 `WorldKernel` 闸门回到中枢**。

### 3.3 turn-scoped layered runtime(模块边界,职责以 world-runtime spec 为准)

TurnOrchestrator · InputRouter · **WorldKernel(唯一 durable 写入者)** ·
ContextAssembler · **PerceptionResolver(唯一喂角色的入口)** · Director · AgentRuntime
· Reactor · Materializer · MemoryBeliefSystem · OffstageReconciler · TasteSeedRuntime ·
StudioRuntime。不建常驻服务端模拟;世界交互时推进,回来懒补(理由=1.6 成本约束)。

### 3.4 全回合流 + 快/慢路径(治延迟)

回合 = 多次串行 LLM 调用跑在用户 key 上,故按「是否阻塞用户读到第一段散文」切分:

- **快路径(阻塞首 token)**:Director 选角 + 被选角色**流式发言** → 用户立刻有东西读。
- **慢路径(发言流式时并行 / 之后结算)**:Reactor 落 delta、记忆、传话、后置 flesh、
  A 的硬状态后检。**结构真相允许比散文稍晚,只要在下一回合构建 context 前落库。**
- **提交屏障**:复用宪章已定的 AI Town 式实例锁——**一次只提交一个回合**;用户可
  立即**读**,但下一回合**提交**需等本回合结构结算完(通常阅读时间 > 慢路径耗时,
  基本无感)。
- **深度档(快省/标准/深)**:调 `maxActiveAgents`、记忆/反思频率、flesh 阈值、A
  后检范围。**MVP 先单档跑通,档位作为紧随其后的成本阀。**

### 3.5 感知投影怎么产生:机械为底 + 导演调显著

「中枢再告诉每个人感知到什么」的落点:

- **事实层** = WorldState 的**确定性过滤**(在场人 / 可见物 / 自己记忆)——**结构上
  不可能发散、便宜**。
- **导演**只在其上调**显著性 / 注意力**(把压力线迹象推到前台、决定「你这才注意到
  角落坐着人」),**绝不发明事实**。

排除「导演逐个生成感知」:那会把感知本身变成生成的,可能重新引入它本要消除的发散,
且每角色一次生成、贵。与 A(下)同哲学:**事实接地,框架/注意力自由。**

### 3.6 God 改硬事实 = delta + 见证作用域 reconcile(改即重算)

God 不打破规则,是「拥有最宽的合法提议权,但提议仍经校验、可审计、可回滚」:

1. God 硬改 → 走 **WorldKernel** 校验落库(provenance `god-edited`);
2. 触发一次 **OffstageReconciler 式一致性修复**,把矛盾的记忆/信念/关系账本拉齐;
3. **覆盖而非删除**(被宪章「追加式日志 / 绝不删记忆」逼出):旧记录标 `superseded`
   + 追加覆盖记录 → Inspector 可显示「第 N 回合被 God 重写」,Fork 可回到 retcon 前;
4. **级联深度 = 见证作用域**:拉齐被点名实体 + 所有「见证过被推翻之事」的角色;
   更深的二手 hearsay 留给自然衰减。一致性守在「能当场拆穿」这一层,代价天然有界。

### 3.7 现状 vs 目标(诚实对账)

`docs/DESIGN.md` 显示**地基已相当完整**(feed/play、BYO-key+IndexedDB、
`WorldRules+WorldState+Delta`、`reactor→validate→apply` 闸门、deltaLog、锁/gate/
portable、lore 注入、主观记忆+反思+关系账本+传话、地点 stub→fleshed、
establishCharacter、evolveWhileAway、重生成),但多数职责仍集中在单体 `runTurn`。

> **这不是从零造,而是「从单体 `runTurn` 向外拆模块」。** 不要另起竞争性循环;按
> world-runtime spec 的 slice 顺序把边界一段段抽出,语义保持不变。

### 3.8 实施顺序(对齐 runtime spec slices)

1. 抽 **WorldKernel**(+ rejected 记录) 2. **ContextAssembler + PerceptionResolver**
(+ 证明 POV 边界的测试) 3. **Director Casting**(active/ambient 上限,删 tension≥6
硬编码) 4. **AgentRuntime** 显式认知步 5. **Memory/Belief** 分型 + provenance
6. **Offstage + Pressure Lines** 7. **Studio instrumentation**。每 slice **先写
证明边界的测试再动实现**。

## Part 4 — 执行态硬问题:已定方向 + 残余权衡

1.6 说约束与美学在内核层同向;下面是它们在**执行层分叉**之处。讨论已给出方向。

### 4.1 A — 渲染态一致性(散文可对状态撒谎)

闸门只保护**落库的 state**,但用户**读到的是散文**。这是「真」的最后一公里。

- **分声部化解大半**(复用「Director 全知 / 角色片面」):
  - **角色声部**(对白/动作主张)→ **允许错**,这正是戏剧;其物理主张**绝不静默变
    state**,被 Reactor 路由成「该角色信念」或「一句谎言」,错得入账。**这与 1.3 的
    发散是同一枚硬币**——防的正是「部分感知者私自把自生成的现实塞进世界」。
  - **旁白 / Director 声部** → **必须接地**;说「门开着」门就得是开的(且该落 delta)。
- **policing = 混合**:日常只靠 **prompt 接地**(硬事实快照喂旁白,零额外调用);
  **仅对硬状态物体**(门/锁/关键道具/在不在场/位置)跑**轻量后检**,不符则标记/
  软修/重生。平衡点:没人因一句闲聊被拆穿,也不为每句描写付费。

### 4.2 B / C — 延迟 + 成本(同一个经济问题)

见 3.4:快/慢路径 + 实例锁屏障 + 深度档。残余权衡:深度档是否进 MVP(建议先单档)。

### 4.3 D — Reactor 门控 + 精度(成本 × 「世界活着」的交点)

- **门控 = 导演顺手判 + 偏向跑**:导演本就在跑、全知,让它多带一个「本回合可能有
  结构变化」标志 → **零额外调用**。**不确定时偏向跑**——「宁可多烧一次,也绝不让
  世界忘事」把头号价值(欠提议=「打翻的杯子下一句就忘」)编码进默认行为。
- **精度**:证据优先 + Voyager 式 **critic**(每条 delta 附散文证据,拒绝记理由);
  高后果 delta 加二次确认,低后果从宽。

### 4.4 Feed 冷启动成本与质量(漏斗最上游,仍largely开放)

生成好门本身是一串 LLM 调用;BYO-key 下「光浏览也要付费/配 key」是获取期摩擦。
方向:**静态示例门池**做足(无 key 首屏惊艳)+ 后台预生成池 + 多样性约束;把
feed 生成质量当成与 runtime 并列的一等子系统,有自己的评测信号(秒级可判率、
推门转化率、扎根率)。**此项尚需一个明确的拥有者与指标。**

### 4.5 频道隔离的长期税

每加一个 power 面都要重交税,且漏隔离的 bug **静默**(世界「不该知道却知道」)。
方向:把「角色 prompt 里绝不出现 directorNote / sceneContract / 跨世界 taste /
未 canon 化 god-edit」做成**常驻测试断言**;**PerceptionResolver 作为唯一喂角色的
入口**,隔离只需在一处守住。
