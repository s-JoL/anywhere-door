# 浮生 / The Reveries — 世界卡"一眼可判" + 行为驱动世界生成（带多样性） 设计

- **日期**: 2026-06-23
- **状态**: 详细设计（待复核 → 分阶段实现）
- **两个问题**：(A) 用户在 feed 里**一眼**就能判断喜不喜欢一个世界——文字世界没有视频那种 0.5 秒多巴胺，必须工程化造出"瞬时信号"。(B) 一个**世界生成模型**按用户历史行为造世界，既贴合喜好、又保持**多样性**避免陷入局部最优（信息茧房）。

---

## Part A — 世界卡的"一秒判断"设计

### 核心难点
TikTok 靠**视频(动+声)**在 0.5s 给信号。文字世界没有。所以卡片不能是"世界说明书"(目录脸)，必须在一屏内造出**瞬时可感的信号**。人对一个故事世界的"想不想进去"，1 秒内其实只看四样：

1. **类型 + 调性**(温柔恋爱?暗黑?情色?悬疑?冒险?)——最大的筛子，必须最快可见。
2. **尺度/烈度**(治愈↔露骨、安全↔危险)——RP 用户极在意。
3. **钩子**(一段**真实世界口吻的冷开场**)——像一本书的第一句话:一句话就知道这声音抓不抓你、有没有张力。这是"那一秒"的真正载荷。
4. **谁在里面**(角色的勾人一行)——人是连接点，要露出角色的悬念/算计。

### 设计原则
- **卡片是"冷开场"，不是简介。** 不写"一个有角色的雨夜酒馆"(目录)，而是直接把你扔进一个**带张力的瞬间**，用第二人称"你"，结尾**悬住**(邀请你行动，而非旁观)。
- **体验领先，标题辅助。** 大字是钩子，标题做小标签。
- **可扫信号分层**:调性 chips + 烈度点 → 一眼归类；钩子 → 决定"这口味对不对"；角色一行 → "这些人有意思吗"。
- **动效即预览**:卡片聚焦时钩子**逐字打出来**(1–2s)，既是 TikTok 式的动感钩子，又**预演了"文字会活过来"这一核心机制**——让用户一眼明白这不是静态故事，是会流动的世界。
- **避开目录陷阱**:不列功能、不堆设定。

### WorldSeed 新增"展示层"字段
```ts
interface WorldSeed {
  // ...既有...
  presentation: {
    genre: string;                 // 武侠 / 黑色 / 科幻 / 情色…(主类型，做主 chip)
    mood: string[];                // 雨夜·危险·孤独(2–3 个调性 chip)
    intensity: "calm" | "charged" | "explicit";  // 烈度点(暖→霓虹桃红渐变)
    hook: string;                  // 冷开场:1–3 句，第二人称，结尾悬住(那一秒的载荷)
    cast: { name: string; line: string }[];  // 每角色一句razor一行:名+身份+一丝算计
    accent?: string;               // 该世界的强调色(主题化卡片，避免千篇一律)
  };
}
```
- 内置/创作世界:作者填(create 表单加这些字段，并给"钩子怎么写"的引导)。
- 导入卡:从 V2 字段派生(genre←tags、hook←first_mes 截取/改写、cast←角色名+description 首句)。
- **AI 生成世界:生成器直接产出这些(尤其 hook——必须被提示词要求写"杀手级第一句")。**

## Part B — 行为驱动的世界生成 + 多样性

### 架构总览（五个部件）
```
用户行为 ──▶ 口味模型 ──▶ 候选生成(检索 + LLM 生成) ──▶ 排序(利用×探索+多样性) ──▶ feed
   ▲                                                                              │
   └──────────────────────── 反馈回路(滑过/停留/扎根) ◀───────────────────────────┘
```

### 1. 行为信号(本地优先，存设备)
按强度加权采集:
- **强正**:扎根(turns≥N / 停留久)、回访、创作/导入某题材、在世界里主动推进露骨/暴力/温情等特定基调。
- **弱正**:进入(点了"推门")。
- **负**:快速划过(停留 < 1.5s 未进入)、进去一两轮就退。
- **In-world 口味**:用户自己常选的动作基调(主动挑衅?调情?探案?)——比"进了哪个世界"更真实地反映口味。

### 2. 口味模型(taste profile)
两层表示:
- **结构化标签权重**:`Record<dimension, weight>`，维度含 genre、mood、intensity、pacing、角色原型(冷艳/憨厚/危险…)、主题(复仇/悬疑/暧昧…)。指数衰减(近期行为权重高)。
- **(可选)嵌入向量**:把"进过/扎根的世界"的 hook+worldview 嵌入，取加权质心做口味向量(P3 上云接 embedding 时启用；本地阶段先用标签)。
- 全程**本地**(契合产品本地优先/隐私)。

### 3. 候选生成(两路)
- **检索**:对已有种子池(内置 + 创作 + 导入 + 未来社区)按口味打分。
- **生成(核心)**:一个 **LLM 世界生成器**，输入 = 口味画像 + 一个**多样性指令** + 创造性温度，输出一个完整 `WorldSeed`(世界观/规则/2 角色/openingState/**presentation(尤其杀手级 hook)**)。提示词强约束:写一句抓人的冷开场、角色要带秘密目标、贴合(或故意偏离)口味维度。
  - **预生成、后台批量**:绝不在"划到时"现生成(慢+贵)。维护一个**预生成候选池**(后台/空闲时用杂活档便宜模型批量造)，feed 实时只做排序(便宜、本地)。

### 4. 排序 —— 利用 × 探索 + 多样性（"避免局部最优"的核心）
纯按"预测喜欢"最大化 = 信息茧房。用三件经典手段组合:
- **利用/探索(explore/exploit)**:大部分(~70–80%)给高口味契合世界(利用);留一部分(ε)给**有意偏离**口味的新世界(探索)。把"维度/题材"当**多臂老虎机(bandit)**的臂，用 **Thompson 采样 / UCB** 平衡"已知爱好的收益"与"未充分探索维度的不确定性"——欠探索的题材会被主动采样，防止口味坍缩。
- **feed 内多样性(MMR, Maximal Marginal Relevance)**:重排时 `score = 相关性 − λ·与已展示世界的相似度`，保证连刷不是"十个雨夜酒馆"。
- **惊喜注入(serendipity)**:周期性塞一个**远离口味**的高新颖世界，主动打破局部最优——并从反应里学习(用户若扎根，说明发现了新口味)。
- **新鲜度/防腻**:降权近期已展示/已进入的；"你最近暗黑世界刷多了，换个亮的"。

### 5. 反馈回路
每次滑过/停留/扎根 → 更新口味标签权重 + bandit 臂的回报。生成的世界被扎根 → 强化其维度组合(并可"再生成相似但更深"的变体)；被快划 → 惩罚。

### 冷启动
新用户无历史 → 给一个**精心策划的多样化起始集**(跨题材的内置世界)+ 几个广谱生成世界；前几次滑动快速建模。

### 成本/延迟
- 生成走**后台 + 杂活档(便宜模型)+ 批量**，维护候选池;feed 排序是本地廉价计算。
- 与"自带 key"一致:生成消耗的是用户/创作者的 key(后台限速)；上云后可由平台补贴热门生成。

## 分阶段实现建议
- **阶段 1(可立即做，低风险)**:`presentation` 字段 + 卡片"冷开场"重设计 + 逐字打入动效 + create 表单/导入派生这些字段 + 给内置世界补写 hook/cast/chips。**这步独立交付，立刻提升 feed 的"一眼可判"。**
- **阶段 2**:行为信号采集(本地事件:enter/dwell/swipe-away/在世界里的 turns)+ 口味标签模型(纯函数，可测)。
- **阶段 3**:排序层(利用打分 + ε-探索 + MMR 多样性 + 防腻)over 现有种子池。bandit 先用简单 ε-greedy，再升 Thompson。
- **阶段 4**:LLM 世界生成器(条件化 + 多样性提示)+ 后台预生成候选池 + 接入 feed。
- **阶段 5(上云后)**:嵌入向量口味、跨用户协同信号、社区世界池。

## 需要确认的决策
1. **阶段 1 先做**(卡片"冷开场"+ presentation 字段)——这步我建议立刻实现，独立见效。其余 2–4 是更大的系统。
2. **生成器的 key/成本**:生成用谁的 key?(建议:用户自带 key、后台空闲时限速预生成 N 个；上云再谈补贴。)
3. **探索比例 ε**:起步给多大?(建议 ~25%，可随"用户是否容易腻"自适应。)
4. **本地 vs 云**:阶段 2–3 纯本地即可;阶段 4 生成也可本地(用户 key)。真正需要云的是跨用户协同 + 社区池(阶段 5)——仍是那个待定的后端决策。
